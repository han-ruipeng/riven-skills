## 概述

BOSS直聘求职全流程助手。从职位筛选、主动沟通、面试安排到 offer 复盘，
覆盖求职全链路。**所有用户特异性参数集中在 `personal-profile.md` §〇**，
skill 逻辑层保持通用。

## 前置规则（每次执行必读）

以下规则在每次处理 JD 时按顺序执行，具体阈值从 `personal-profile.md` §〇 取值。

### 薪资过滤
- JD 薪资范围上限 < `personal-profile.md §〇.最低月薪` → ❌ 不满足门槛，跳过。
- JD 无薪资信息 → 先进行投递，时机差不多时向boss询问。

### 简历选择
- JD 薪资范围上限 ≥ `personal-profile.md §〇.简历.自动选择规则` 中的阈值 → 使用简历A
- 否则 → 使用简历B
- 打招呼消息中不提及简历文件名，统一说"我的简历"。

### 地点匹配
- JD 工作地点在首选城市列表 → 🟢 优先投递
- JD 工作地点在次选城市列表 → 🟡 可以投递
- 其他城市 → 询问用户

### 行业匹配
- JD 所属公司匹配优先行业列表 → 🟢 优先投递
- 匹配次选行业 → 🟡 可以投递
- 匹配排斥行业 → ❌ 直接过滤
- 其他 → 正常，询问用户

## 模块详解

### 模块 1：JD 快速筛选

> **目的**：数秒判断一个职位是否值得投递。

**执行步骤**：
1. 提取 JD 中的：岗位名称、薪资范围、工作地点、公司名称/行业。
2. 按「前置规则」逐项比对，阈值来自 `personal-profile.md` §〇。
3. 输出判定结果 + 推荐使用的简历。

**输出模板**：
```
| 检查项 | 结果 |
|--------|------|
| 薪资 | XX-XXK → 满足/不满足 → 使用简历A/B |
| 地点 | XX → 🟢首选 / 🟡次选 / ⚠️需确认 |
| 行业 | XX → 🟢优先 / 🟡次选 / ❌排斥 |
| 岗位 | XX → 匹配 / 不匹配 |

✅/❌ 综合：建议投递 / 建议跳过
```

---

### 模块 2：主动沟通

> **目的**：根据 JD 和用户画像生成个性化打招呼消息。

**触发**：用户粘贴职位链接/截图/描述，或说"帮我想一句打招呼的话"。

**输入**：
- 目标岗位的 JD（职位描述全文或关键句）
- 结合 `personal-profile.md` 中的个人亮点（技能栈、项目经历）

**输出**：
- 一段 50-80 字的打招呼消息
- 2 个备选版本（数据冲击版 & 技术匹配版）

**Boss直聘对话限制**：
- 首次主动联系时，对方未回复前只能发 1 条消息
- 字数限制约 200 字，建议 50-80 字

---

### 模块 3：JD 深度解读

> **目的**：深度解析职位描述。

**触发**：用户粘贴 JD 文本，说"分析一下这个岗位"。

**输出维度**：
| 维度 | 分析内容 |
|------|---------|
| 🎯 核心需求 | 岗位最看重的 3 个能力 |
| 📌 必备技能 | 明确要求的技术栈/经验 |
| ✨ 加分项 | 锦上添花但不必须的 |
| ⚠️ 可能坑点 | JD 中模糊/矛盾的描述、变相加班信号 |
| 💡 简历对策 | 针对此 JD 应突出哪些内容、用哪份简历 |
| 🔑 关键词 | 刷简历筛选的关键词清单 |

---

### 模块 4：公司调研

> **目的**：在决定投递前快速评估公司。

**触发**：用户给出公司名称。

**执行步骤**：
1. 用已知信息评估公司背景、融资阶段、规模、口碑。
2. 建议用户去"看准网"、"脉脉"、"天眼查"交叉验证。
3. 输出风险评估：🟢绿灯 / 🟡黄灯 / 🔴红灯。

---

### 模块 5：面试安排与准备

> **目的**：管理面试排班 + 生成面试题库。

**触发**：用户收到面试邀请，说"帮我安排面试" / "帮我准备面试"。

---

#### 5.1 面试形式

根据 `personal-profile.md` §〇「面试.支持线下城市」判断：

- 面试所在城市 ∈ 线下城市白名单 → 可由用户选择线上或线下
- 面试所在城市 ∉ 线下城市白名单 → 仅接受线上
- 如果对方要求线下但你不在该城市，礼貌说明"目前不在该城市"，争取远程

---

#### 5.2 面试排班（铁律）

从 `personal-profile.md` §〇「面试.排班铁律」获取规则，通常为：
- 每半天最多 1 场面试
- 已排定时段自动锁定
- 线上+线下不安排在同一天
- 同一公司多轮尽量隔天
- 线下预留通勤缓冲

排班表维护：`interview-schedule.md`。

---

#### 5.3 面试准备内容

收到面试邀请后，结合 `personal-profile.md` 的常问答案例（§五）和 JD 生成：

1. **必问题**：自我介绍、离职原因、期望薪资、职业规划 → 结合用户真实经历
2. **技术题**：根据 JD 技能要求和用户技能栈（§二）出题
3. **项目深挖**：针对用户项目（§四）设计追问链（STAR框架）
4. **反问HR**：5 个展现思考深度的反问问题
5. **可选**：模拟面试角色扮演

---

### 模块 6：进度跟踪

> **目的**：用 `tracker.md` 记录所有投递和面试状态。

**数据字段**：
```
| 日期 | 公司 | 岗位 | 薪资范围 | 使用简历 | 城市 | 行业 | 状态 | 备注 | 下次跟进 |
```

**使用方式**：用户随时说"更新进度 / 看进度"，Claude 读写 `tracker.md`。

---

### 模块 7：薪资谈判

> **目的**：在 HR 面或谈 offer 时提供谈判策略。

**触发**：用户收到 offer 或进入薪资沟通环节。

核心策略见 `templates/salary-negotiation.md`。

## 注意事项

1. **模块化设计**：所有用户特异性参数集中在 `personal-profile.md` §〇，他人使用时只需修改该文件。
2. **简历管理**：简历已在 BOSS 直聘账号中，根据薪资阈值自动选择。
3. **隐私保护**：`personal-profile.md` 包含个人信息，切勿上传到公开仓库。
4. **真实性**：所有生成内容基于用户真实经历，切勿虚构。
5. **平台规则**：本 skill 只做"人的辅助"，不涉及任何自动化操作。
6. **模板适配**：`templates/` 下的模板包含示例关键词，使用时结合 personal-profile 个性化。
